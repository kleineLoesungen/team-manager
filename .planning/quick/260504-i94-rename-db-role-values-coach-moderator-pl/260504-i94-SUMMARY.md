## Migration Strategy

Two-path migration for zero-downtime safety:

1. **Standalone SQL** (`migrate_004_rename_roles.sql`): For production DBs already deployed. Idempotent — detects whether old roles exist before acting, recreates all affected RLS policies in the same transaction.

2. **Auto-migration in PHP** (`maybe_migrate_db()` Migration 004): Runs automatically on first HTTP request. Detects old role values, renames them, drops/recreates all role-dependent RLS policies. Safe to run on already-migrated DBs (detection query short-circuits).
