<objective>
Fix two bugs:
1. Nested `<form>` elements in the free list save-cells table cause the browser to fold the delete-row form's hidden inputs into the outer save-cells form. Clicking Speichern submits `action=delete_row` (the last duplicate value wins) with no `confirm`, which triggers the deletion confirmation screen instead of saving.
2. Three checkboxes (`show_all_rows` in list_form.php; `coach_only` in both free list and member list "Lokale Spalte hinzufügen" forms in list_detail.php) should be Bootstrap `form-switch` toggle switches, consistent with the cell-level boolean toggles already present in both files.

Purpose: Restores correct save behaviour for free lists; improves visual consistency of boolean controls.
Output: Two PHP template files with no nested forms and toggle switches in place of checkboxes.
</objective>

<task type="auto">
  <name>Task 1: Fix nested form bug in free list table (list_detail.php)</name>
  <files>src/templates/coach/list_detail.php</files>
  <action>
In the free list `else` branch (lines 159–266), the outer `<form method="POST" action=save_cells>` contains per-row delete-row forms inside each `<td>` in the Aktion column (around line 212). This is invalid HTML.

Fix:
1. For each row in `$free_rows`, declare a standalone delete form OUTSIDE (before) the save-cells form. Give it `id="delete-row-{row_id}"` and place it immediately above the save-cells `<form>` opening tag (i.e. before line 159). Use a PHP foreach to emit all delete forms before the save form opens.
   Each delete form looks like:
   ```php
   <form id="delete-row-<?= (int)$row['id'] ?>" method="POST"
         action="/moderator/lists/<?= (int)$list['id'] ?>">
       <?= csrf_field() ?>
       <input type="hidden" name="action" value="delete_row">
       <input type="hidden" name="row_id" value="<?= (int)$row['id'] ?>">
   </form>
   ```
2. Inside the table's Aktion column `<td>`, replace the entire nested `<form>...</form>` block with a single button that references the external form via the HTML5 `form` attribute:
   ```php
   <button type="submit" form="delete-row-<?= (int)$row['id'] ?>"
           class="btn btn-sm btn-outline-danger">Löschen</button>
   ```
3. Do NOT change the `elseif (empty($columns))` branch (lines 131–156) — those delete forms are standalone and are not nested inside a save form; leave them exactly as-is.
4. Do NOT change any other logic, totals calculation, or member list section.
  </action>
