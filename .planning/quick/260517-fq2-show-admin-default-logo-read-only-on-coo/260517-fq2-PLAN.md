<task type="auto">
  <name>Task 2: Add logo img to navbar before app title</name>
  <files>src/templates/layout.php</files>
  <action>
In render_navbar() in layout.php, the current brand span is:
```php
<span class="navbar-brand fw-semibold"><?= $brand ?></span>
```

Replace it with a span that includes an inline img before the text. The img uses `1lh` (1 line-height unit) for height — this matches the current font size exactly. Add `onerror="this.style.display='none'"` as a safety fallback so a broken img doesn't leave an empty gap (even though /logo returns 404 naturally, the onerror cleanup is cleaner UX):

```php
<span class="navbar-brand fw-semibold d-flex align-items-center gap-2">
    <img src="/logo" alt="" aria-hidden="true"
         style="height:1lh; width:auto; object-fit:contain;"
         onerror="this.style.display='none'">
    <?= $brand ?>
</span>
```

Use `height:1lh` — this is a CSS unit supported in all modern browsers meaning "1 line height", which naturally matches the font-size of the surrounding text. The `alt=""` with `aria-hidden="true"` marks it as decorative. The `onerror` hides the img element if /logo 404s so no broken image icon appears.

Do NOT change anything else in render_navbar() or render_layout_head(). The favicon link `<link rel="icon" href="/logo">` already exists in render_layout_head() — do not duplicate it.
  </action>
  <verify>Load any page in the app (admin, coordinator, member, login). The navbar should show the logo image inline before the app title text. If no logo is set, the image is invisible (hidden by onerror). The layout should not break on mobile — d-flex align-items-center keeps everything on one line.</verify>
  <done>Logo img appears before brand text in navbar across all roles and layouts. Missing logo causes no visible broken image. Alignment is inline with the text on the same line.</done>
