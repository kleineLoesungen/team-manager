   ```php
   function maybe_migrate_db(PDO $pdo): void {
       $schema = preg_replace('/[^a-zA-Z0-9_]/', '', DB_SCHEMA);
       // Migration 001: coach_only flag on columns
       $pdo->exec(
           "ALTER TABLE {$schema}.columns
            ADD COLUMN IF NOT EXISTS coach_only BOOLEAN NOT NULL DEFAULT FALSE"
       );
       // Migration 001: update columns_visibility_select RLS to respect coach_only
       // DROP + recreate is idempotent (IF EXISTS guard)
       $pdo->exec("DROP POLICY IF EXISTS columns_visibility_select ON {$schema}.columns");
       $pdo->exec("CREATE POLICY columns_visibility_select ON {$schema}.columns FOR SELECT USING (
           current_setting('app.is_admin', true) = 'true'
           OR (current_setting('app.current_role', true) = 'coach'
               AND team_id = NULLIF(current_setting('app.current_team_id', true), '')::integer)
           OR (list_id IS NULL
               AND team_id = NULLIF(current_setting('app.current_team_id', true), '')::integer)
           OR (list_id IS NOT NULL
               AND coach_only = FALSE
               AND team_id = NULLIF(current_setting('app.current_team_id', true), '')::integer
               AND EXISTS (SELECT 1 FROM {$schema}.lists
                           WHERE lists.id = columns.list_id
                           AND lists.visibility IN ('public', 'protected')))
       )");
   }
   ```
   Note: `maybe_migrate_db` runs on EVERY boot — all statements must be idempotent. `ADD COLUMN IF NOT EXISTS` and `DROP POLICY IF EXISTS` + recreate satisfy this.

In `src/player/list_detail_handler.php`, update the column SELECT query (`$col_stmt`) to exclude `coach_only = TRUE` local columns.

Current query has:
```sql
SELECT c.id, c.name, c.data_type, c.list_id
FROM columns c
WHERE c.is_active = TRUE
  AND (
    c.list_id = ?
    OR (c.list_id IS NULL AND c.team_id = ?
        AND EXISTS (
          SELECT 1 FROM list_global_columns lgc
          WHERE lgc.list_id = ? AND lgc.column_id = c.id
        ))
  )
ORDER BY (c.list_id IS NULL) DESC, c.sort_order, c.created_at
```

Change the `c.list_id = ?` branch to `(c.list_id = ? AND c.coach_only = FALSE)`:
```sql
SELECT c.id, c.name, c.data_type, c.list_id
FROM columns c
WHERE c.is_active = TRUE
  AND (
    (c.list_id = ? AND c.coach_only = FALSE)
    OR (c.list_id IS NULL AND c.team_id = ?
        AND EXISTS (
          SELECT 1 FROM list_global_columns lgc
          WHERE lgc.list_id = ? AND lgc.column_id = c.id
        ))
  )
ORDER BY (c.list_id IS NULL) DESC, c.sort_order, c.created_at
```

Global columns (`list_id IS NULL`) do not have a `coach_only` concept (they are always visible to players); only local columns need this filter. The RLS policy is defense-in-depth — this PHP-level filter is the authoritative application check.
