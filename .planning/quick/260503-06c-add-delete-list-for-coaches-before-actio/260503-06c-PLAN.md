<task type="auto">
  <name>Task 1: Add delete handler (two-step POST flow)</name>
  <files>src/coach/list_delete_handler.php</files>
  <action>
Create `src/coach/list_delete_handler.php`. This handler is reached via POST only.

Flow:
1. Call `require_coach()` and `require ROOT_PATH . '/src/templates/coach/layout.php'`
2. Reject non-POST requests with `redirect('/coach/lists')`
3. Call `require_csrf()`
4. Extract `$list_id = (int)($_REQUEST['list_id'] ?? 0)` and `$confirm = (int)($_POST['confirm'] ?? 0)`
5. Fetch list from DB: `SELECT id, name FROM lists WHERE id = ? AND team_id = ?` — redirect to `/coach/lists` if not found (ownership check, defense-in-depth)
6. If `$confirm !== 1`: render a confirmation page using `render_coach_page()` — no delete yet.
   The page shows:
   - Heading: "Liste löschen"
   - Warning card (Bootstrap `alert alert-danger`): "Liste **{name}** wirklich löschen? Alle Daten (Spalten, Einträge) werden **unwiderruflich** gelöscht."
   - A POST form to `/coach/lists/{id}/delete` with `csrf_field()`, hidden `<input type="hidden" name="confirm" value="1">`, a red submit button "Endgültig löschen" (`btn btn-danger min-touch`), and a cancel link back to `/coach/lists/{id}/settings` (`btn btn-outline-secondary ms-2 min-touch`).
7. If `$confirm === 1`: execute `DELETE FROM lists WHERE id = ? AND team_id = ?` — the DB cascades remove all columns, cells, and list_global_columns entries automatically. Then `redirect('/coach/lists')`.

Use the same PDO/error_log pattern as the rest of the codebase. Wrap the DELETE in try/catch and render an error via `render_coach_page()` on PDOException rather than crashing.

Do NOT use JavaScript at any point.
  </action>

  </verify>
  <done>File exists, no syntax errors, implements both branches (confirm=0 shows warning page; confirm=1 deletes and redirects)</done>
</task>
