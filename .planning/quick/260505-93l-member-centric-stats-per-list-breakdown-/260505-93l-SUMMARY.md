# Quick Task 260505-93l: Member-Centric Stats Per-List Breakdown Summary

**One-liner:** Per-list breakdown table on both stats pages — members see their own list history, moderators inspect any member via a no-JS GET dropdown.

## What Was Built

### Task 1 + 2: Member stats per-list breakdown (`/member/stats`)

Three new queries added after the existing aggregation block in `src/player/stats_handler.php`:

- **Query A:** `list_global_columns` JOIN to find lists with global columns attached, visibility-filtered to `public` + `protected`
- **Query B:** cells for the logged-in member across those lists, reshaped into `$per_list_cells[list_id][column_id]`
- **Query C:** total list counts per column via `list_global_columns` (boolean % denominator)
- PHP loop builds `$per_list_totals` (sum for number, count_true for boolean)

New "Listenübersicht" table in `src/templates/player/stats.php`:
- Only renders when `$per_list_rows` is non-empty
- Sorted date DESC NULLS LAST, undated at bottom
- Boolean cells: ✓ / ✗; null cells: —
- `<tfoot>` Gesamt row: integer/decimal sum for numbers, `count (pct%)` for booleans

### Task 3: Moderator per-list breakdown (`/moderator/stats`)

Handler additions in `src/coach/stats_handler.php`:
- Fetches `$all_members` (active members, sorted first+last name) for the dropdown
- Parses + validates `$_GET['member_id']` against known member IDs
- When member selected: runs same three queries without visibility filter (moderator sees all lists)
- Passes 8 new variables into the template `use` clause

Template additions in `src/templates/coach/stats.php` (inside the `else` block, after ranking table):
- `<hr>` + "Listenübersicht" heading
- GET form with member `<select>` + "Anzeigen" button; hidden inputs preserve existing filter state
- "Zurücksetzen" link (shown only when member selected) strips `member_id` while keeping other filters
- Per-list table identical in structure to member version
- Empty state: text message when selected member has no lists with global columns
