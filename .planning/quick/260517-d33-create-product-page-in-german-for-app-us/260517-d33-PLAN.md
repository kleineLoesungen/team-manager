<task type="auto">
  <name>Task 1: Create product.html — German landing page</name>
  <files>product.html</files>
  <action>
Create a single self-contained HTML file at the project root: `product.html`. No build step, no external file dependencies beyond CDN links.

Design guidelines:
- Load Bootstrap 5.3 via CDN (same SRI hash pattern used in the app)
- Use Google Fonts (Inter or similar sans-serif) via CDN link tag
- Color palette: dark navy (#0f172a) as primary dark, white, and one accent color (e.g. indigo #6366f1 or a clean blue)
- Sections (in order):
  1. **Navbar** — minimal, sticky. Logo text "Team Manager" left-aligned. No links needed (standalone page).
  2. **Hero** — Large headline + subline + one primary CTA button ("Jetzt kostenlos starten" or similar placeholder href="#"). Background: subtle gradient or dark background with a light abstract SVG shape/blob. Headline example: "Dein Team. Deine Listen. Deine Statistiken."
  3. **Features** — 3-column grid (stacks to 1 col on mobile). Three feature cards with icon (Bootstrap Icons via CDN), bold title, and 1-2 sentence description. Features:
     - "Flexible Listen" — Eigene Spalten für jede Liste, genau so wie du es brauchst
     - "Mitglieder tragen selbst ein" — Mitglieder pflegen ihre Daten direkt, du behältst den Überblick
     - "Statistik auf einen Blick" — Listenübergreifende Auswertung pro Mitglied, filterbar nach Zeitraum
  4. **Social Proof / Quote** — Single centered pull-quote block. Example: "Endlich weiß ich auf einen Blick, wer wie oft mitgemacht hat." — Eine Koordinatorin
  5. **CTA Section** — Centered, contrasting background (accent color). Headline: "Bereit loszulegen?" + button.
  6. **Footer** — Single row, centered. "© 2025 Team Manager · Alle Rechte vorbehalten"

Inline `<style>` block for custom CSS (no separate .css file). Use CSS custom properties for colors. Smooth scroll behavior on html element. Cards should have subtle border-radius (12px) and box-shadow. Hero section should be at least 100vh. Section padding: py-5 Bootstrap classes.

Bootstrap Icons CDN: `https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.3/font/bootstrap-icons.min.css`

The page must work when opened directly as a file (file:// protocol) — no server required.

<task type="auto">
  <name>Task 2: Exclude product.html from deploy.sh</name>
  <files>deploy.sh</files>
  <action>
Edit deploy.sh to add `--exclude='product.html'` to the first `mirror --reverse` command (the one that mirrors the repo root to public_html/team-manager/).

Insert the new exclude line directly after the existing `--exclude='README.md' \` line (before the `. public_html/team-manager/` destination line) so it is alphabetically grouped with the other root-level file exclusions.

Resulting block (abbreviated):
```
mirror --reverse \
    --exclude-glob='.DS_Store' \
    --exclude='.git/' \
    --exclude='.planning/' \
    --exclude='docker/' \
    --exclude='docker-compose.yml' \
    --exclude='.env' \
    --exclude='.env.docker' \
    --exclude='.env.example' \
    --exclude='config.php' \
    --exclude='deploy.sh' \
    --exclude='README.md' \
    --exclude='product.html' \
    . public_html/team-manager/
```

Do not change anything else in deploy.sh.
  </action>
  <verify>grep "product.html" deploy.sh returns the exclude line</verify>
  <done>deploy.sh contains `--exclude='product.html'` in the first mirror command</done>
