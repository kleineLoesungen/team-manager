## What Was Built

### Task 1 — app_color Setting (DB + Handler)
- `db_init_schema()` seeds `app_color = '#2563eb'` alongside `app_title` on fresh installs
- Migration 003 in `maybe_migrate_db()` seeds the default on existing databases (idempotent via `ON CONFLICT DO NOTHING`)
- `settings_handler.php` saves `app_color` on POST with `#rrggbb` hex validation (fallback `#2563eb`)
- Settings form adds a color picker (`<input type="color">`) synced to a text field via minimal inline JS
- GET path loads `app_color` from DB and passes it into the render closure

### Task 2 — Brand Color + Modern CSS (layout.php)
- `render_layout_head()` fetches `app_color` from the `settings` table using a `static` PHP variable cache (single DB query per request)
- CSS `:root` block defines `--brand` and overrides `--bs-primary`, `--bs-btn-bg`, `--bs-btn-border-color` so Bootstrap components pick up the brand color automatically
- Navbar stripped of `navbar-light bg-white border-bottom` — CSS rule sets `background-color: var(--brand)` with white text
- Cards: `border-radius: 0.75rem` + `box-shadow: 0 1px 3px rgba(0,0,0,.08)`
- Tables: vertical cell borders removed via `border-right: none; border-left: none` — horizontal borders only
- Active sidebar/tab links: `background-color: var(--brand)` with white text
- `btn-primary` and `btn-outline-primary` both respect `--brand`; hover uses `color-mix()` for darkening

### Task 3 — Boolean Form-Switch (coach/list_detail.php)
- Boolean cells wrapped in `.form-check.form-switch.mb-0` div — renders as Bootstrap pill toggle
- `role="switch"` added for accessibility semantics
- Form submission behavior unchanged (value=1 for checked, absent for unchecked)
- Total/count row for boolean columns unaffected
